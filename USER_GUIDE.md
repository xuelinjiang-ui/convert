# 用户使用说明

## 1. 程序用途

本程序用于将以下音频文件转换为 `mp3`：

- `.flac`
- `.ogg`
- `.mgg`
- `.mflac`
- `.ncm`
- `.kgg`
- `.kgma`
- `.kwm`

输出文件默认保存到当前用户桌面，文件名与原文件同名，仅扩展名变为 `.mp3`。

## 2. 运行前提

### 2.1 系统要求

- Windows 系统
- 已安装 `.NET Framework 4.7.2`

### 2.2 程序目录要求

程序运行目录中需要包含以下文件：

- `WpfApp1.exe`
- `Tools\ffmpeg.exe`

对于部分加密格式，还需要以下附带文件存在于程序输出目录：

- `QQMusicDecryptRunner.exe`
- `QQMusicDecryptHook.dll`
- `EasyHook.dll`
- `EasyHook32.dll`
- `EasyHook64.dll`
- `EasyLoad32.dll`
- `EasyLoad64.dll`
- `KugouInfraRunner.exe`
- `System.Buffers.dll`
- `System.Memory.dll`
- `System.Runtime.CompilerServices.Unsafe.dll`
- `System.Numerics.Vectors.dll`

## 3. 各格式使用前提

### 3.1 `.flac` / `.ogg`

无需额外客户端环境，程序会直接调用 `ffmpeg` 转码。

### 3.2 `.ncm`

无需安装网易云音乐客户端，程序可在本地直接解码。

### 3.3 `.kwm`

无需安装酷我音乐客户端，程序可在本地直接解密。

### 3.4 `.kgma`

无需调用外部解密 EXE，程序已内置本地解密逻辑。

### 3.5 `.kgg`

需要满足以下条件：

- 本机已安装酷狗音乐
- 系统中存在酷狗安装目录
- 存在 `infra.dll`
- 存在酷狗数据库 `KGMusicV3.db`
- 目标文件最好已在酷狗中播放过一次，以便数据库中存在对应密钥

### 3.6 `.mgg` / `.mflac`

需要满足以下条件：

- 本机已安装 QQ 音乐
- QQ 音乐可正常启动
- 程序目录中带有 `QQMusicDecryptRunner.exe` 和相关 EasyHook 文件

## 4. 使用步骤

### 4.1 启动程序

双击运行 `WpfApp1.exe`。

### 4.2 选择文件

在界面中选择需要转换的音频文件。

### 4.3 开始转换

程序会自动判断文件格式，并执行以下流程之一：

- 直接转码
- 先解密再转码
- 先解码成明文音频，再转为 `mp3`

### 4.4 获取结果

转换成功后，输出文件会生成在桌面。

## 5. 输出规则

- 输出格式固定为 `.mp3`
- 输出文件名为原文件名
- 中间解密文件会先写入系统临时目录
- 正常结束后临时目录会自动清理

## 6. 常见问题

### 6.1 提示未找到 `ffmpeg.exe`

原因：

- 程序运行目录缺少 `Tools\ffmpeg.exe`

处理方式：

- 将 `ffmpeg.exe` 放到 `Tools` 目录下

### 6.2 提示未找到解密输出文件

原因：

- 上游解密流程没有成功落盘
- 目标客户端未正确返回结果

处理方式：

- 确认本机相关音乐客户端可正常启动
- 重新尝试转换

### 6.3 `.kgg` 提示未找到密钥

原因：

- 酷狗数据库中没有该文件对应的密钥记录

处理方式：

- 先使用酷狗音乐播放该文件一次，再重新转换

### 6.4 `.mgg` / `.mflac` 转换失败

原因：

- QQ 音乐未安装
- QQ 音乐未启动且自动启动失败
- 注入相关文件缺失

处理方式：

- 确认 QQ 音乐已安装
- 确认程序目录内附带了 Runner、Hook、EasyHook 文件

### 6.5 含中文路径时 FFmpeg 报错

程序已经内置路径中转机制，会自动使用 ASCII 临时目录转换。如果仍失败，通常说明上游解密文件本身没有成功生成。

## 7. 适用范围说明

本程序适用于本地个人环境中，针对以下格式进行转换：

- 明文音频：`flac`、`ogg`
- 网易云加密音频：`ncm`
- QQ 音乐加密音频：`mgg`、`mflac`
- 酷狗加密音频：`kgg`、`kgma`
- 酷我加密音频：`kwm`

对于依赖客户端密钥或客户端内部 DLL 的格式，如果缺少对应客户端环境，程序无法完成转换。

## 8. 适配性评估

在满足本文档前提条件的情况下，各格式在普通用户电脑上的适配性并不完全相同。

### 8.1 适配性较高

- `flac`
- `ogg`
- `ncm`
- `kwm`
- `kgma`

原因：

- 这几类格式主要依赖本程序内置逻辑或 `ffmpeg`
- 对用户电脑上的第三方播放器内部 DLL、数据库结构依赖较少
- 只要程序目录完整、系统环境正常，通常成功率较高

### 8.2 适配性中等

- `kgg`

原因：

- 依赖酷狗安装环境
- 依赖 `infra.dll`
- 依赖 `KGMusicV3.db`
- 依赖数据库中已经存在目标文件对应密钥

说明：

- 程序已经通过 `KugouInfraRunner.exe` 解决了 64 位主程序与 32 位 `infra.dll` 的兼容问题
- 但如果用户电脑上的酷狗版本差异较大，或数据库中没有对应密钥，仍可能转换失败

### 8.3 适配性相对较低

- `mgg`
- `mflac`

原因：

- 依赖 QQ 音乐客户端安装与运行状态
- 依赖进程注入成功
- 依赖 QQ 内部解密接口和 DLL 行为没有变化
- 兼容性容易受到 QQ 版本、安全软件、系统环境影响

### 8.4 使用建议

如果你准备把程序提供给其他用户使用，可以按以下预期说明：

- `flac / ogg / ncm / kwm / kgma`：兼容性较高
- `kgg`：需要酷狗环境完整，且最好先在酷狗中播放过目标文件
- `mgg / mflac`：需要 QQ 音乐环境可用，兼容性受用户机器差异影响最大
